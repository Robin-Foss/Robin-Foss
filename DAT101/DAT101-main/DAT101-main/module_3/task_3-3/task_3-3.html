    2. [Creating a function is about defining what the function does and how it behaves and invoking a function is about running the function to perform its defined task.]

  2. [Explain the syntax regarding semicolons and code blocks.]

    3. [Semicolons use them to terminate statements to avoid errors and code blocks uses curly braces {} to group statements together ]

  3. [What happens when we try to store values from a function when it doesn't return any?]

    4. [Default return value if a function doesn’t have a return statement, it returns undefined by default and storing the result when you store the result of such a function, the variable will hold undefined.]

  4. [How do we separate parameters from each other when creating a function?]

    5. [when creating a function, you separate parameters using commas. Parameters are listed within the parentheses () following the function name.]

  5. [Explain how parameter names are related during the invocation and creation of a function.]

    6. [When you create a function, you define its parameters. These parameters act as placeholders for the values that will be passed to the function when it is invoked.]

  6. [Explain why a parameter is undefined when a function call is missing a parameter.]

    7. [when a function is called without providing all the expected parameters, the missing parameters are assigned the value undefined. This happens because JavaScript does not enforce a strict number of arguments for function calls.]

  7. [Explain what happens when we use recursive functions]

    8. [Recursive functions are functions that call themselves in order to solve a problem. This technique is useful for problems that can be broken down into smaller, similar sub-problems. However, it’s important to ensure that the recursion has a base case to prevent infinite loops and potential stack overflow errors.]
